# TH-INVENTORY-PDA 배포 가이드

## 프로젝트 개요

TH-INVENTORY-PDA는 티에이치스터디 재고관리 PDA 프로젝트입니다.

React + TypeScript + Vite 기반으로 개발되었으며 AWS Lightsail Ubuntu, Nginx, Cloudflare 환경에서 운영합니다.

```text
사용자
→ Cloudflare
→ AWS Lightsail Ubuntu
→ Nginx
→ /var/www/th-inventory-pda
→ React Build 파일
```

현재 서비스 구성

```text
티에이치스터디 본 사이트
https://www.th-study.com

재고관리 PDA
https://pda.th-study.com
```

# 로컬 빌드

이미 개발 서버가 정상 실행되고 있고 node_modules 폴더가 존재한다면 npm install은 다시 실행하지 않아도 됩니다.

```bash
npm run build # dist 생성
```

최초 설치 환경인 경우

```bash

npm run build # dist 생성
```

```text
dist
├── assets
├── favicon.svg
├── icons.svg
└── index.html
```

# Nginx 설정 생성

```bash
sudo nano /etc/nginx/sites-available/th-inventory-pda # 설정 생성
```

기본 설정

```nginx
server {
    listen 80;
    server_name pda.th-study.com;

    root /var/www/th-inventory-pda;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|svg|ico|webp)$ {
        expires 30d;
        access_log off;
    }
}
```

React SPA는 새로고침 시 404 방지를 위해 반드시 아래 설정이 필요합니다.

```nginx
try_files $uri $uri/ /index.html;
```

---

# Nginx 활성화

sites-available은 설정 보관소입니다.

sites-enabled는 실제 사용중인 설정입니다.

```bash
sudo ln -s /etc/nginx/sites-available/th-inventory-pda /etc/nginx/sites-enabled/th-inventory-pda # 최초 1회
```

설정 검사

```bash
sudo nginx -t # 설정 검사
```

반영

```bash
sudo systemctl reload nginx # 설정 반영
```

확인

```bash
ls -al /etc/nginx/sites-enabled # 활성화 확인
```

정상 결과

```text
syntax is ok
test is successful
```

# Cloudflare DNS 설정

Cloudflare DNS에 PDA 서브도메인을 추가합니다.

```text
Type
A

Name
pda

Content
[AWS Lightsail 공인IP]

Proxy Status
Proxied

TTL
Auto
```

설정 예시

```text
pda.th-study.com
A
[AWS Lightsail 공인IP]
Proxied
Auto
```

주황 구름(Proxied)이 활성화되어 있어야 Cloudflare를 통해 접속됩니다.

# pda.th-study.com 인증서 발급

Certbot Nginx 플러그인을 사용하면 인증서 발급과 Nginx 설정이 자동으로 처리됩니다.

```bash
sudo certbot --nginx -d pda.th-study.com # 인증서 발급 및 Nginx 자동 설정
```

정상 완료 예시

```text
Successfully received certificate
Successfully deployed certificate
Congratulations! You have successfully enabled HTTPS
```

인증서 확인

```bash
sudo certbot certificates # 인증서 확인
```

---

# Certbot이 자동으로 수행하는 작업

아래 작업을 자동으로 처리합니다.

```text
도메인 검증
Let's Encrypt 인증서 발급
443 HTTPS 설정 생성
80 → 443 리다이렉트 생성
Nginx 설정 수정
Nginx Reload
```

따라서 PDA 신규 배포 시 HTTPS 적용은 대부분 아래 명령어 하나로 완료됩니다.

```bash
sudo certbot --nginx -d pda.th-study.com
```


```nginx

```

기존 티에이치스터디 설정에 PDA 설정을 섞지 않습니다.

```text
th-study.com
www.th-study.com
↓
티에이치스터디 본 사이트

pda.th-study.com
↓
재고관리 PDA
```

# 재배포 방법

React 프로젝트 수정 후에는 서버를 중지할 필요가 없습니다.

다시 빌드합니다.

```bash
npm run build # dist 재생성
```

파일질라에서 기존 파일 덮어쓰기

```text
TH-INVENTORY-PDA/dist/*

↓

/var/www/th-inventory-pda/
```

React 정적 파일만 변경한 경우

```text
서버 재시작 불필요
Nginx 재시작 불필요
파일 덮어쓰기만 수행
```

Nginx 설정을 변경한 경우에만

```bash
sudo nginx -t # 설정 검사
sudo systemctl reload nginx # 설정 반영
```

실행합니다.
